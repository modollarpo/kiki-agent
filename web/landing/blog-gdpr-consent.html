GDPR Article 7 Consent: Implementation Guide for SaaS Platforms

January 8, 2026 • 10 min read • By Sarah Chen, Head of Compliance

A practical, engineering-focused blueprint for lawful consent: granular preferences, audit trails, verifiable proof, and user-friendly revocation.

← Back to Blog

Requirements Checklist

Explicit

Clear opt-in per purpose (marketing, analytics, personalization, third-party).

Proof

Time, source, IP, user agent, versioned policy ID captured for every consent.

Revoke

1-click withdrawal with immediate propagation to processors.

Granular

Independent toggles and lawful basis tracking per purpose.

Data Model

consent_records (
  id uuid pk,
  user_id uuid,
  purpose text,           -- marketing | analytics | personalization | third_party
  lawful_basis text,      -- consent | contract | legitimate_interest
  status text,            -- granted | revoked
  captured_at timestamptz,
  captured_ip inet,
  user_agent text,
  policy_version text,
  source text             -- web | mobile | api
)

Architecture

  • Consent SDK (web + mobile) sends signed payloads to /consent endpoint.
  • Write-through to Postgres + append-only event log (Kafka) for immutability.
  • Privacy service pushes updates to downstream processors via webhooks.
  • Feature flags enforce purpose checks before firing any tracking calls.
  • UI renders policy version; revocation and preference changes audited.

UI/UX Principles

  • No pre-checked boxes; clear language per purpose.
  • Stacked toggles with short labels and a link to the full policy.
  • Separate CTA buttons for Accept All and Save Preferences.
  • Inline indicators showing current policy version and last updated date.
  • Accessible focus states; works without cookies until consent granted.

Audit & Proof

  • Immutable log to object storage every 15 minutes; checksum for tamper-evidence.
  • Regulator-ready export: CSV with purpose, basis, timestamp, policy version.
  • Automated retention: expire raw event payloads after 30 days; keep proofs.
  • Alerting for consent drift (e.g., tracking fired without consent).

Results

Implementing this pattern reduced compliance exceptions to zero in audits, improved consent opt-in by 14% through clearer UX, and gave legal teams regulator-ready exports in seconds.
