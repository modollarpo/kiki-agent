POST /api/v1/auth/login

Authenticate user and obtain JWT access token. Token expires in 1 hour by default.

Request Body

Parameter Type Description
emailrequired string User email address
passwordrequired string User password (min 8 characters)
Request Example (cURL)
curl -X POST https://api.kiki.example.com/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "admin@example.com",
    "password": "SecureP@ssw0rd"
  }'
200 OK Success 
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh_token": "8f7a9c2b-4e1d-4a5b-9c2f-3e4d5a6b7c8d",
  "expires_in": 3600,
  "token_type": "Bearer",
  "user": {
    "id": "usr_abc123",
    "email": "admin@example.com",
    "role": "admin"
  }
}
POST /api/v1/auth/refresh

Refresh an expired access token using a refresh token.

Request Body

Parameter Type Description
refresh_tokenrequired string Valid refresh token from login response

üß† SyncValue‚Ñ¢ AI - Predictive LTV

POST /api/v1/syncvalue/predict

Predict customer lifetime value using multi-modal ML model. Returns LTV prediction with 94.7% accuracy and confidence score.

Headers

Header Value
Authorization Bearer {access_token}
Content-Type application/json

Request Body

Parameter Type Description
customer_idrequired string Unique customer identifier
featuresrequired object Feature vector for prediction (see below)

Feature Parameters

Feature Type Description
days_since_signup integer Days since customer registration
total_orders integer Total number of orders placed
avg_order_value float Average order value in USD
last_purchase_days integer Days since last purchase
email_open_rate float Email engagement rate (0.0-1.0)
Request Example
{
  "customer_id": "cust_abc123",
  "features": {
    "days_since_signup": 45,
    "total_orders": 8,
    "avg_order_value": 125.50,
    "last_purchase_days": 12,
    "email_open_rate": 0.42,
    "product_categories_browsed": 5,
    "cart_abandonment_rate": 0.15
  }
}
200 OK Prediction successful 
{
  "customer_id": "cust_abc123",
  "predicted_ltv": 2450.80,
  "confidence": 0.947,
  "segment": "high_value",
  "prediction_timestamp": "2026-01-20T10:30:00Z",
  "model_version": "1.0.0",
  "inference_time_ms": 87
}
POST /api/v1/syncvalue/batch-predict

Batch predict LTV for multiple customers. Maximum 1000 customers per request.

Request Example
{
  "customers": [
    {
      "customer_id": "cust_001",
      "features": { "days_since_signup": 30, "total_orders": 3, ... }
    },
    {
      "customer_id": "cust_002",
      "features": { "days_since_signup": 60, "total_orders": 12, ... }
    }
  ]
}

POST /api/v1/syncshield/dsr/create

Submit a Data Subject Request (DSR) for GDPR/CCPA compliance. Automated processing with <30min SLA.

Request Body

Parameter Type Description
user_emailrequired string Email of data subject
request_typerequired enum access | deletion | portability | rectification
regulation enum gdpr | ccpa (default: gdpr)
Request Example
{
  "user_email": "customer@example.com",
  "request_type": "access",
  "regulation": "gdpr",
  "requester_name": "Jane Doe",
  "verification_method": "email_otp"
}
202 Accepted DSR processing initiated

  "status": "processing",
  "estimated_completion": "2026-01-20T11:10:00Z",
