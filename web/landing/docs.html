Quick Start Guide

Get up and running with KIKI in under 15 minutes. This guide covers the essential steps to deploy your first retention campaign.

Prerequisites

  • Kubernetes Cluster: v1.24+ (AWS EKS, GKE, or AKS recommended)
  • Helm: v3.8+ installed and configured
  • kubectl: CLI access to your cluster
  • Docker Registry: Access to a container registry (ECR, GCR, DockerHub)
  • Domain Name: For production deployments
  • SSL Certificate: TLS certificate for HTTPS (Let's Encrypt or ACM)

Step 1: Clone the Repository

Terminal
git clone https://github.com/your-org/kiki-platform.git
cd kiki-platform

Step 2: Configure Environment

Copy the example configuration and customize for your environment:

Terminal
cp charts/kiki/values-example.yaml charts/kiki/values-prod.yaml
# Edit values-prod.yaml with your settings

‚ö†Ô∏è Security Notice

Never commit sensitive values like API keys or passwords to version control. Use Kubernetes Secrets or External Secrets Operator.

Step 3: Install Dependencies

Terminal
# Install PostgreSQL (if not using external DB)
helm repo add bitnami https://charts.bitnami.com/bitnami
helm install kiki-postgres bitnami/postgresql \
  --set auth.database=kiki \
  --set auth.username=kiki \
  --set auth.password=CHANGE_ME

# Install Redis
helm install kiki-redis bitnami/redis \
  --set auth.password=CHANGE_ME

Step 4: Deploy KIKI Platform

Terminal
helm install kiki ./charts/kiki \
  -f charts/kiki/values-prod.yaml \
  --namespace kiki \
  --create-namespace

Step 5: Verify Deployment

Terminal
# Check all pods are running
kubectl get pods -n kiki

# Expected output (6 agents):
# NAME                           READY   STATUS    RESTARTS
# syncvalue-xxx                  1/1     Running   0
# syncengage-xxx                 1/1     Running   0
# syncshield-xxx                 1/1     Running   0
# syncflow-xxx                   1/1     Running   0
# synccreate-xxx                 1/1     Running   0
# billing-xxx                    1/1     Running   0

Step 6: Access the Platform

Get the external IP or domain of the ingress controller:

Terminal
kubectl get ingress -n kiki
# Navigate to https://your-domain.com

Your KIKI platform is now running. Next steps: Configure your first integration and create a retention campaign.

Installation Options

Production Deployment (Recommended)

For production workloads, we recommend the following architecture:

  • Multi-AZ Kubernetes: Minimum 3 nodes across 3 availability zones
  • Managed Databases: AWS RDS PostgreSQL (Multi-AZ) or equivalent
  • Managed Cache: AWS ElastiCache Redis (cluster mode) or equivalent
  • Load Balancer: Application Load Balancer with SSL termination
  • Monitoring: Prometheus + Grafana + AlertManager
  • Logging: ELK Stack or CloudWatch Logs

Development/Staging

For testing and development, use Docker Compose for rapid local iteration:

docker-compose.yml
docker-compose up -d
# Access at http://localhost:8080

Authentication

KIKI uses JWT-based authentication with role-based access control (RBAC).

POST /api/v1/auth/login

Obtaining an Access Token

Request Body (JSON)
{
  "email": "admin@example.com",
  "password": "your-secure-password"
}
Response (200 OK)
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh_token": "8f7a9c2b...",
  "expires_in": 3600,
  "token_type": "Bearer"
}

Using the Access Token

Include the token in the Authorization header for all API requests:

Platform Architecture

KIKI follows a microservices architecture with 6 specialized agents communicating via gRPC and event-driven messaging.

System Components

  • API Gateway: Kong or NGINX Ingress Controller with rate limiting
  • Service Mesh: Istio (optional) for advanced traffic management
  • Message Bus: Apache Kafka for event streaming
  • Cache Layer: Redis for session state and temporary data
  • Primary Database: PostgreSQL for transactional data
  • Analytics DB: Snowflake or BigQuery for data warehouse
  • Object Storage: S3 for media assets and backups

Data Flow

  1. Ingestion: Customer events arrive via webhooks or SDK
  2. Processing: SyncValue analyzes LTV, SyncEngage scores churn risk
  3. Decision: Retention triggers evaluated based on business rules
  4. Execution: SyncFlow orchestrates multi-channel campaigns
  5. Compliance: SyncShield validates all actions against policies
  6. Monitoring: All actions logged to dual audit trail

GDPR Implementation

KIKI provides built-in GDPR compliance with automated DSR (Data Subject Request) handling.

Consent Management (Article 7)

POST /api/v1/consent/record
Request
{
  "user_id": "user_xyz789",
  "consent_type": "marketing",
  "granted": true,
  "consent_method": "web_form",
  "ip_address": "203.0.113.42",
  "user_agent": "Mozilla/5.0..."
}

Data Subject Requests (Articles 15-22)

Automated DSR processing with <30-minute SLA:

  • Right to Access (Art. 15): Complete data export in JSON format
  • Right to Erasure (Art. 17): Cascading deletion across all systems
  • Right to Portability (Art. 20): Machine-readable export
  • Right to Rectification (Art. 16): Data update API
